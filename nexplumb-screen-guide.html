What Is Nexplumb?

Nexplumb is a two-sided online marketplace that connects customers in Nigeria (starting in Lagos) with verified, trusted artisans — plumbers, electricians, carpenters, painters, and other tradespeople. Think of it like a Nigerian version of Uber, but for skilled home services. A customer opens the app, finds a plumber, pays safely, watches them arrive on a live map, and reviews the job afterwards.

The entire web application is made up of four separate portals — each designed for a different type of user — all sharing the same brand, design system, and backend. This document walks you through every screen in all four portals, telling you exactly what belongs on each page, why it is there, and how it should work.

How to use this guide: Use the left sidebar to jump to any screen. Each screen card tells you its purpose, lists every section that must appear on that page, and explains what each element does in plain language. Designers should read this before opening Figma. Developers should reference it alongside the UX document.

Design System Basics — Colours, Fonts & Rules

Before we describe any screen, you need to understand the shared visual language. Every screen across all four portals uses these same colours, fonts, and spacing rules. Consistency is a trust signal for users.

Font: Arial is used for all text at this stage. Body text must never be smaller than 14px. All button labels are written in sentence case (like this) — never ALL CAPS.

Buttons: The primary "Book Now" style button is orange (#E76F51), white text, 48px tall, 8px rounded corners. The secondary outlined button has a navy border, navy text, transparent background. On hover it fills navy with white text.

Breakpoints (Responsive Design): Every screen must work at three sizes — Desktop (1280px wide), Tablet (768px wide), and Mobile Web (375px wide). Design desktop first, then simplify for smaller sizes.

🏠 Customer Web Portal
CW-1

Public Homepage / Landing Page

URL: nexplumb.com  ·  Accessible by anyone, logged in or not
Purpose of This Screen This is the first thing any visitor sees when they type nexplumb.com or arrive from a Google search. It must immediately answer three questions in the visitor's mind: "What is this?", "Can I trust it?", and "How do I get started?". It is also the most important page for Google Search ranking (SEO), so every piece of text must be written with that in mind.
⚠️
Design Rule: The homepage serves two masters — Google's search crawler and a first-time Nigerian customer who is skeptical. Every section must earn its place. No filler copy. The search bar is the single most important element on the page.
Section 1 — Global Navigation Bar (Sticky Top)
🔝
Navigation Bar — Always Visible

This bar sits at the very top of the page and never disappears as the user scrolls down. It "sticks" to the top of the browser window at all times. This is called a "sticky" or "fixed" navbar.

  • Left side: The Nexplumb logo in navy colour. Clicking it always takes the user back to the homepage from any page on the site.
  • Centre links: Four navigation links — "How It Works", "Find an Artisan", "For Artisans", and "Pricing". These are text links, not buttons. They link to different sections of the site.
  • Right side: A "Log In" text link (for returning users) and a "Get Started" button styled in orange (the primary brand colour). This orange button is the most important action in the navbar.
  • Extra link: "Join as an Artisan" — a separate link (can be smaller, different colour) for tradespeople who want to sign up. Clicking this goes to the Artisan Registration page.
  • On scroll behaviour: When the user starts scrolling down the page, the navbar background turns white with a subtle shadow underneath. This makes it readable against any page content below. Before scrolling, it appears transparent over the dark hero section.
Section 2 — Hero Section (Main Banner)
🌟
Hero — The First Thing Eyes Land On

The "hero" is the large banner at the very top of the page, below the navbar. It takes up the full width of the screen and is usually 500–600px tall on desktop. The background is Nexplumb Navy (#0D2137). This is where visitors decide in seconds whether to stay or leave.

  • Main headline (H1): Large text that says something like "Your trusted artisan, one tap away". This must include the words "Lagos" and "Nigeria" somewhere visible on the hero for SEO purposes. Font must be bold, white, 44px minimum on desktop.
  • Sub-headline: A shorter line below the headline explaining who the platform is for and where. Example: "Find verified plumbers, electricians, and tradespeople near you in Lagos — instantly." This is white text, smaller than the headline, about 18–20px.
  • Search Bar: This is the most important element on the entire page. It is a large white box, full-width within a white card, with placeholder text: "What do you need? (e.g. leaking pipe, electrical fault...)". Clicking it or pressing Enter takes the user to the Search Results page.
  • Location Selector: Directly below the search bar — a field that either auto-detects the user's area (using their browser location) or lets them type an area name or LGA manually (e.g. "Yaba", "Surulere").
  • Service Category Chips: Below the location field, a row of clickable pills/tags: "Plumbing · Electrical · Carpentry · Painting · Tiling". Each one is a quick shortcut — clicking "Plumbing" takes the user directly to filtered search results for plumbers.
  • Hero Illustration: On the right side of the hero (desktop only), an illustration showing a Nigerian home, an artisan, and the Nexplumb shield/logo. This makes the page feel local and relevant, not like a generic foreign tech product.
Section 3 — Trust Bar
✅
Trust Metrics — 3 Key Statistics

Immediately below the hero, a full-width dark band (same navy background or slightly darker) shows three important statistics side by side. These numbers build immediate trust with a skeptical visitor.

  • Stat 1: "1,200+ Verified Artisans" — with a person/shield icon. This shows there are real people on the platform.
  • Stat 2: "Escrow-Protected Payments" — with a padlock icon. This addresses the Nigerian customer's biggest fear: losing money to fraud.
  • Stat 3: "4.8 / 5 Average Rating" — with a star icon. This shows quality and customer satisfaction.
  • Layout: Three equal columns on desktop. Stack vertically on mobile. Each has: an icon, a bold large number/phrase, and a short label below.
Section 4 — How It Works
📋
3-Step Process Explanation

This section explains the booking process in the simplest possible way. Many visitors will be first-time users of any artisan platform. This section removes fear by showing how simple it is.

  • Section title: "Book a trusted artisan in 3 steps" — centred, bold, large.
  • Step 1 — Search: A circle with "1" inside, an icon, bold title "Search", and a 2-line description. Example: "Type what you need or pick a category. We'll show you verified artisans near your location."
  • Step 2 — Book & Pay Safely: Circle "2", icon, title, description. "Choose your artisan, pick a time, and pay through our secure escrow. Your money is protected until the job is done."
  • Step 3 — Track & Confirm: Circle "3", icon, title, description. "Watch your artisan travel to you on a live map. Confirm when the job is complete and release payment."
  • CTA below the steps: An orange "Find an artisan now" button. This gives the visitor a second chance to start searching after reading how it works.
Section 5 — Featured Artisans
👷
Top-Rated Artisans Near You

A section that shows real artisans from the platform. This makes the site feel alive and trustworthy — not just a landing page with no actual service behind it.

  • Title: "Top-rated artisans near you" with the user's location label (e.g. "in Lagos") shown next to it.
  • Artisan Cards: A row of 4 cards on desktop (scrollable horizontally on mobile). Each card shows: artisan profile photo (circular), name, trade (e.g. "Plumber"), star rating, number of completed jobs, area/location, and a "View Profile" button.
  • "See all artisans" link: At the end of the row, a text link that goes to the full search results page.
Section 6 — Category SEO Sections
🔍
Trade-Specific Sections (Critical for Google)

These sections are one of the most important parts of the homepage from an SEO perspective. When someone Googles "plumber in Lagos" or "electrician near me in Nigeria", these sections are what helps Nexplumb appear in results.

  • One section per trade category: "Plumbers in Lagos", "Electricians in Lagos", "Carpenters in Lagos", etc.
  • Each section contains: A heading using the trade and city name (H2 tag for SEO), 3 featured artisan cards from that category, and a "Find more plumbers near you" CTA link.
  • Why this matters: Google indexes text on the page. By writing "Plumbers in Lagos" as a real heading with real content beneath it, Nexplumb can rank when someone searches that phrase.
Section 7 — Social Proof / Customer Reviews
⭐
Real Customer Testimonials

Three review cards from real customers who have completed bookings. This is called "social proof" — seeing other people's positive experiences makes new visitors trust the platform.

  • Each review card shows: Customer photo (or avatar), first name only, their area (e.g. "Lekki"), the star rating they gave, and 2–3 sentences of their written review.
  • Important rule: All reviews must come from real completed jobs. No made-up quotes.
Section 8 — Footer
🔗
Site Footer — Bottom of Every Page

The footer appears at the bottom of every customer-facing page. It has a dark navy background and is divided into four columns of links.

  • Column 1 — About Nexplumb: Links to About Us, How It Works, Blog, and Press.
  • Column 2 — For Customers: Links to Find an Artisan, How Escrow Works, Safety Guarantee, and Pricing.
  • Column 3 — For Artisans: Links to Join as an Artisan, How to Get Verified, Artisan Resources, and Training.
  • Column 4 — Support: Links to Help Centre, Contact Us, Report an Issue, and FAQs.
  • Social Icons: Row of icons for Twitter, Instagram, LinkedIn, Facebook, and YouTube.
  • Legal Links: Small text at the very bottom — "Terms of Service · Privacy Policy · Escrow Policy".
  • WhatsApp Floating Button: A green WhatsApp icon that floats in the bottom-right corner of the screen on every page. Clicking it opens a WhatsApp chat with the Nexplumb support team. This is an extremely important trust feature for Nigerian users who are very comfortable with WhatsApp.
CW-S

Customer Sign Up Page

URL: nexplumb.com/register  ·  Triggered when clicking "Get Started" or "Book" without an account
Purpose This is the page where a new customer creates their Nexplumb account. The goal is to make this as fast and frictionless as possible. Requiring too many fields at sign-up causes people to leave. We collect only what is essential.
Layout
📝
Two-Panel Layout (Desktop)
  • Left panel (40% width): A navy background panel showing the Nexplumb logo, a brief statement like "Join thousands of Lagos residents who book trusted artisans safely", and the three trust stats (verified artisans, escrow protection, average rating). This panel reassures users that signing up is worthwhile.
  • Right panel (60% width): White background containing the actual sign-up form. On mobile, only the form is shown (the left panel is hidden).
Sign-Up Form Fields
✍️
Form Elements (Keep It Short)
  • Page title: "Create your account" — bold, 24px, navy colour.
  • First Name field: Label above the field reads "First Name". Required. 48px tall input box.
  • Phone Number field: Label reads "Phone Number". Required. Show Nigeria flag + "+234" prefix automatically. This is the main login identifier — no email required for basic accounts.
  • Password field: Label reads "Create a password". A small eye icon on the right lets the user toggle to show/hide what they're typing. Minimum 8 characters. A small progress bar below shows password strength (weak/moderate/strong).
  • Confirm Password field: "Repeat your password". Shows a green checkmark inline when both passwords match.
  • Terms checkbox: "I agree to Nexplumb's Terms of Service and Privacy Policy." The words "Terms of Service" and "Privacy Policy" are clickable links. The checkbox must be ticked before the form can be submitted.
  • Submit button: Full-width orange button, "Create Account". Shows a loading spinner while the account is being created.
  • OTP Verification step: After clicking "Create Account", the form slides to a new screen asking the user to enter the 6-digit code sent to their phone via SMS. Auto-reads on Android. "Resend code" link appears after 60 seconds.
  • "Already have an account?" link: Below the button. Links to the Login page.
  • Social divider: "— or continue with —" between the form and a Google sign-in button (optional for Phase 2).
💡
Beginner tip: Every form field needs a visible label above it — never just placeholder text that disappears when you start typing. This is both an accessibility requirement and a usability standard.
CW-L

Customer Login Page

URL: nexplumb.com/login
Purpose Returning customers log in here to access their dashboard, manage bookings, and start new bookings. It must be fast — a returning user should be back in their account within 10 seconds.
Login Form
🔐
Login Fields
  • Phone Number field: Same as sign-up — "+234" prefix, Nigerian phone format.
  • Password field: With show/hide toggle icon.
  • "Forgot password?" link: Right-aligned above the submit button. Clicking this starts the password reset flow: enter phone number → receive OTP via SMS → set new password.
  • Submit button: Full-width orange "Log In" button.
  • Error state: If credentials are wrong, a red banner appears below the button saying: "Incorrect phone number or password. Please try again." — Never say which field is wrong (this is a security practice).
  • "Don't have an account?" link: Below button. Links to Sign Up page.
CW-2

Search Results Page

URL: nexplumb.com/search?q=plumber&location=yaba&sort=rating
Purpose The customer has typed what they need and pressed search. This page shows them all matching artisans with tools to narrow down their choice. It must balance showing enough information to make a decision, without being overwhelming.
Desktop Layout — Two Columns
📐
Left Panel — Filter Sidebar (280px wide)

A panel fixed to the left side of the screen that allows the user to narrow their search results. It scrolls independently from the results on the right.

  • "Clear all filters" link: At the very top of the sidebar. Resets all filters to their defaults and re-runs the search.
  • Applied Filters Chips: Below the clear link, any active filters appear as small dismissible tags (e.g. "Rating: 4+" with an × to remove). This shows users what filters are currently active.
  • Trade Category (checkboxes): A list of trades — Plumbing, Electrical, Carpentry, Painting, Tiling — with checkboxes. Users can select multiple. Checking or unchecking instantly updates the results on the right.
  • Minimum Rating (star selector): Radio buttons or clickable stars — "3 stars and above", "4 stars and above", "5 stars only".
  • Availability toggle: Three options — "Available Now", "Available Today", "Any Time".
  • Distance Slider: A draggable slider from 1km to 20km — "Show artisans within [X] km of me".
  • Price Range (dual-handle slider): Two draggable handles — set a minimum and maximum price. Shows "₦0 — ₦50,000+" with live price labels on the handles.
  • Verified Only toggle: A simple on/off switch — when on, shows only NIN-verified artisans.
  • Certification Level: "Any", "Nexplumb Certified", "Trade Tested" — radio buttons.
📋
Right Panel — Results List (Main Area)
  • Results count & location: At the top, text reads "Showing 24 plumbers in Yaba and nearby areas." This tells the user exactly what they're seeing.
  • Sort dropdown: A dropdown on the right side of the results header with options: "Best match", "Highest rated", "Nearest to me", "Most jobs completed", "Lowest price".
  • View toggle (List / Map): Two buttons at the top-right of the results area — a list icon and a map icon. Clicking Map switches to the Map View described below.
  • Artisan Card (Horizontal, List View): Each artisan appears as a wide horizontal card. From left to right: Profile photo (80×80px circle), then in the centre: Name, trade, star rating with number of reviews, location tags, "Available Now" green badge (if applicable), a 2-line bio, and skill chips (e.g. "Burst pipes · Bathroom installation"). On the right side: price range (e.g. "₦8,000–₦15,000") and two buttons — "View Profile" (secondary, outlined) and "Book Now" (primary, orange).
  • Verified badges on cards: Small inline badges — a shield icon for "ID Verified" and a star icon for "Nexplumb Certified". Shown directly on the artisan card so users don't have to open the profile to see trust signals.
  • Pagination: Shows 24 artisans per page. At the bottom, either a "Load more" button OR numbered page buttons (1, 2, 3...).
🗺️
Map View (Toggle)

When the user clicks the map icon, the layout changes to a 50/50 split. The left half shows an interactive map. The right half shows the scrollable artisan list.

  • Map: Full-height embedded map using Google Maps or OpenStreetMap. Each available artisan appears as an orange pin with their photo thumbnail. Busy artisans appear as gray semi-transparent pins.
  • Customer location: A distinct blue pin with "You" label marks the customer's current location.
  • Hovering a list card: The corresponding artisan pin on the map highlights (grows slightly or changes colour).
  • Clicking a map pin: Opens a small popup "mini-card" above the pin showing: artisan name, photo, rating, trade, price range, and a "View Profile" button.
  • "Search this area" button: Appears when the user pans/drags the map to a new area. Clicking it re-runs the search to find artisans in the newly visible area.
  • Available artisan pins: Animate with a subtle 2-second pulse to draw attention.

Artisan Public Profile Page

URL: nexplumb.com/artisans/[artisan-slug]  ·  Also indexed by Google
Purpose This is where the customer makes their final decision. After browsing search results, they click on one artisan and this page must convince them to book. Think of it as each artisan's personal "store page." It is also indexed by Google, so someone searching for a specific artisan's name should find this page.
Desktop Layout — Two Columns
👤
Left Column (700px) — Profile Content
  • Hero banner image: A wide, full-width image at the top showing the artisan's best job photo or a professional image of them working. Their circular profile photo is overlaid on the bottom-left corner of this banner.
  • Name, trade, and location: Directly below the banner — "Chukwuemeka Okonkwo · Plumber · Surulere, Lagos" in large, bold text.
  • Trust Badges Row: A horizontal row of coloured badge icons: "ID Verified" (blue shield), "Nexplumb Certified" (teal star), "Guarantor Verified" (green check), "Trade Tested" (orange ribbon). Each badge has a tooltip on hover that explains what it means.
  • Stats Row: Four numbers in a horizontal row — large star rating (e.g. "4.8 ★"), number of reviews (e.g. "47 reviews"), jobs completed (e.g. "134 jobs"), member since date (e.g. "Member since Jan 2024").
  • About Section: The artisan's written biography. 2–4 paragraphs describing their background, expertise, and what makes them the best choice. Minimum 150 words encouraged.
  • Skills & Services: A row of pill/chip tags listing specific things this artisan can do: "Burst pipes · Bathroom installation · CCTV drain inspection · Boiler repair · Tap replacement".
  • Portfolio Grid: A 3-column masonry grid (Pinterest-style) of before/after job photos. Clicking any photo opens a full-screen lightbox to see it larger.
  • Service Area Map: A small embedded map showing the artisan's coverage zone — highlighted in teal colour showing which areas of Lagos they serve.
  • Reviews Section: Each review shows: customer first name + their area + date of review + star rating + review text. A bar chart shows the distribution of 1-to-5 star ratings. "See all reviews" link expands to a paginated full list.
📌
Right Column (400px) — Sticky Booking Widget

This panel stays fixed on the right side as the user scrolls through the profile (on desktop). It follows the user so the "Book" button is always visible.

  • Price range: "₦8,000 – ₦15,000 per job" in large bold text. Labelled as "estimated" so the user knows this is a guide, not a fixed quote.
  • Availability: A green "Available Now" badge, or if not available immediately, "Available from [date]".
  • "Book This Artisan" button: The primary, full-width orange button. This is the most important button on the page. Clicking starts the booking flow (CW-4). If not logged in, shows a login prompt first.
  • "Request a Quote" link: Below the main button. For complex jobs where the customer wants a price estimate first. Opens a quote request form.
  • Save to Favourites: A heart icon button. Requires login. Saves the artisan to the customer's "Saved Artisans" list for easy future access.
  • Share button: A share icon with options to "Copy link" and "Share via WhatsApp" — pre-fills a WhatsApp message.
  • Escrow Trust Message: A shield icon with the text "Your payment is protected by Nexplumb Escrow" — in small teal text. Constantly reminds the customer their money is safe.
  • "Message Artisan" button: Opens an in-app chat window. Available only after booking is confirmed, so it cannot be used to bypass the platform. Before booking, a note explains: "You can message after booking."
CW-4

Web Booking Flow (Steps A–D)

URL: nexplumb.com/book/[artisan-slug]
Purpose The booking flow is a multi-step process that takes the customer from "I want to book this artisan" to "My booking is confirmed and paid". It is divided into 4 clear steps. Each step is on its own screen/view. The customer should never feel lost — they always know which step they're on and how many remain.
ℹ️
Step Indicator: At the top of every step, a horizontal progress bar shows the 4 steps: Job Details → Schedule → Review & Pay → Confirmed. The current step is highlighted in teal. Completed steps show a navy checkmark. Upcoming steps are gray. This appears on Steps A, B, C, and D.
Step A — Job Details (CW-4A)
📝
Describe the Job
  • Artisan summary card (persistent left panel): Shows the artisan's photo, name, trade, and rating. This stays visible on the left throughout the booking so the customer remembers who they're booking.
  • Job Type Selector: A dropdown menu listing specific job types: "Pipe repair", "Drain unblocking", "Full bathroom installation", "Tap replacement", "Other (describe below)". Selecting a type pre-fills a suggested description.
  • Job Description text area: A multi-line text box where the customer describes the problem in their own words. Minimum 30 characters required. A live character counter shows how many characters have been typed (e.g. "47 / 30 minimum").
  • Photo Upload: A drag-and-drop zone with a dashed border that says "Drag photos here or click to browse files". Accepts up to 5 photos. As photos are uploaded, thumbnail previews appear in a row below with an × button to remove each one. Helps the artisan understand the job before arriving.
  • Urgency Toggle: A toggle switch labelled "This is urgent (30% premium applies)". When switched on, an explanation appears: "Urgent bookings attract available artisans faster but add a 30% fee to the final total." The toggle must be very clear — customers need to understand the cost implication.
  • Address input: A searchable address field with auto-complete powered by Google Places. As the customer types their address, suggestions appear in a dropdown list. They can also pin a location on a small map preview.
  • "Continue" button: Full-width orange button at the bottom. Goes to Step B. Disabled until required fields are filled.
Step B — Schedule (CW-4B)
📅
Choose Date and Time
  • Full Calendar View: A complete monthly calendar — not a dropdown date picker. The full month grid is shown, like a wall calendar. Available dates (when the artisan is free) are highlighted in teal. Dates in the past and dates when the artisan is fully booked appear gray and are not clickable.
  • Time Slot Grid: When the customer clicks a date, a grid of available hour-blocks appears below the calendar. For example: "8:00 AM · 9:00 AM · 10:00 AM · 11:00 AM · 1:00 PM · 3:00 PM". Each slot is a clickable button. Unavailable slots are gray with a label like "Artisan booked".
  • "Flexible" shortcut: A prominent option above the calendar: "Flexible — any time today or tomorrow (priority matching)". Customers who choose this get faster matching and are shown to more artisans, but pay a small priority premium.
  • Estimated duration: Below the time slots, a note like "Pipe repair jobs typically take 1–2 hours." This helps the customer plan their day.
  • "Back" and "Continue" buttons: Back returns to Step A. Continue moves to Step C (disabled until a date and time are selected).
Step C — Review & Pay (CW-4C)
💳
Payment Step
  • Full job summary: A read-only summary box at the top showing everything the customer entered: artisan name, trade, job description, address, chosen date, chosen time. This is the "review" before payment.
  • Price breakdown table: A clear table showing: "Service estimate: ₦10,000 · Platform booking fee: ₦500 · Materials (if requested): ₦0 · Urgent premium (if selected): ₦3,000 · Total: ₦13,500". Every line item is labelled and explained — no hidden fees.
  • Escrow explanation banner: A full-width banner with a large shield graphic. Text: "Your payment is held safely by Nexplumb and only released to the artisan after you confirm the job is done. Full refund if the artisan does not show up." This banner is critically important for first-time customers.
  • Payment Method Tabs: Four tabs the customer can switch between — "Debit Card", "Bank Transfer", "USSD Code", "Wallet Balance".
  • Debit Card tab: Inline Paystack payment form — card number, expiry, CVV. No redirect to another website. All within the Nexplumb page.
  • Bank Transfer tab: Shows a Nexplumb escrow account number and a unique reference code. Text: "Transfer exactly ₦13,500 to this account. Your booking will be confirmed automatically when payment is received." A "Payment confirmed" status updates live on screen.
  • USSD tab: Shows the USSD code the customer can dial on their phone (e.g. *737*[amount]*[ref]# for GT Bank). Step-by-step instructions for each major bank are shown.
  • Wallet Balance tab: Shows the customer's Nexplumb wallet balance. "Use ₦13,500 from your wallet balance of ₦20,000." Instant confirmation.
  • "Confirm & Pay ₦13,500" button: Full-width orange button. The total amount is shown in the button text so the customer sees exactly what they're paying. Below the button: a padlock icon and the text "Secured by Nexplumb Escrow and Paystack."
Step D — Booking Confirmed (CW-4D)
🎉
Success Screen
  • Animated checkmark: A large teal animated checkmark that draws itself on screen. Below it: "Booking Confirmed!" in large navy bold text.
  • Booking Reference Number: A unique code (e.g. "NX-2024-00847") shown in large monospace font. A "Copy" button lets the customer copy it to their clipboard. This reference is how they track or discuss the booking with support.
  • Summary card: Shows artisan photo, name, job type, date, and time — a neat summary of everything that was just booked.
  • "Track Your Artisan" button: Primary orange button. Takes the customer to the Live Tracking page (CW-5) immediately.

WhatsApp Bot Flow

PurposeA conversational booking experience entirely within WhatsApp. For customers who cannot or prefer not to download the app, and for artisans with limited smartphone data. WhatsApp is used by 95%+ of Nigerian internet users daily — meeting customers where they already are is a massive advantage.
Customer Booking Flow via WhatsApp
  1. Customer sends any message to Nexplumb's WhatsApp number. The bot responds with a greeting: "Welcome to Nexplumb! I can help you book a verified artisan. Reply with what you need — e.g. 'plumber' or 'electrician'."
  2. Customer replies with their trade need (e.g. "plumber"). Bot asks: "Great! Which area of Lagos are you in?"
  3. Customer replies with area (e.g. "Yaba"). Bot shows top 3 available artisans in that area with name, rating, price range, and a numbered choice. "Reply '1', '2', or '3' to see more."
  4. Customer picks an artisan (e.g. "1"). Bot sends the artisan's mini-profile and asks: "When do you need them? Reply with a time or say 'Now'."
  5. Customer selects a time. Bot sends a booking summary and a Paystack payment link. "Tap this link to pay securely via card, transfer, or USSD. Your money is protected until the job is done."

  • New job alert: Artisan receives a WhatsApp message: "New job near you! Plumbing in Yaba · ₦8,000–₦12,000. Reply YES to accept or NO to pass."
  • If YES: Bot sends the customer's area and time. "Job confirmed. Customer is at [area]. Show up by [time]. Safe travels!"

📋 Design Standards
STD-1

Five Required Screen States

Every screen across all portals must be designed in all 5 states
Why This MattersReal users encounter every possible situation — slow internet, empty accounts, error messages, successful completions. If a screen is only designed for the "perfect" state, it will look broken or confusing in every other situation. Every screen must handle all 5 states.
StateWhen It OccursWhat It Must Show
DefaultNormal loaded state with real dataThe primary design — always the first to complete. Shows real representative content, not placeholder "Lorem ipsum" text.
LoadingWhile data is being fetched from the serverSkeleton loaders — gray animated placeholder shapes that match the shape of the real content. Never a blank white page. Never just a spinning circle by itself.
EmptyNo data to display (new user with no bookings, search with no results, cleared filters)An illustration relevant to the context + a descriptive headline + a primary CTA button. Example: empty bookings tab shows an illustration of a calendar + "No bookings yet" + "Find an artisan" button. Never just "No data found."
ErrorServer error, network lost, or invalid data returnedA red banner at the top of the relevant section OR an inline red border on the problematic field. The error message must describe specifically what went wrong. A "Try again" button is always included. Never just "Something went wrong."
SuccessAfter an action completes — booking confirmed, payment sent, review submitted, profile updatedA teal animated checkmark, a summary of what just happened, and a clear next step offered to the user. The user should never have to guess whether their action worked.
STD-2

Accessibility Requirements

Non-negotiable standards for all screens
Why This MattersAccessibility is not optional. It ensures Nexplumb can be used by people with visual impairments, colour blindness, motor limitations, or those who navigate by keyboard. It is also a legal standard (WCAG AA compliance).
  • Colour contrast: All body text must have a minimum 4.5:1 contrast ratio against its background. Large headings: minimum 3:1. Use a contrast checker tool to verify every colour combination.
  • Keyboard navigation: Every interactive element (buttons, links, form fields, dropdown menus) must be reachable and operable using only the keyboard (Tab key to move, Enter to click, Space for checkboxes).
  • Focus ring: Every interactive element must show a visible focus indicator when selected via keyboard — a 2px solid teal (#2A9D8F) outline with 2px offset. Never remove the default focus ring without replacing it with a custom one.
  • Alt text: Every image must have a descriptive alt text attribute. Decorative images (purely visual, no information) should have an empty alt="" attribute so screen readers skip them.
  • Form labels: Every input field must have a visible <label> element associated with it — not just placeholder text. Placeholder text disappears when typing begins and is never a substitute for a label.
  • Error messages: Must describe the problem in text — never use colour alone to indicate an error. "This field is required" in red text is correct. Just turning the field border red is not enough.
  • Maps alternative: Every map view must have a "Switch to list view" button accessible to users without JavaScript or GPS access. Maps should never be the only way to access information.
  • Minimum touch target size: All clickable/tappable elements must be at least 48×48px. Small icon buttons should have invisible padding to reach this minimum.
  • Minimum text size: No body text smaller than 14px on web. No captions smaller than 12px.
  • Reduced motion: All animations must be turned off or reduced when the user has "reduce motion" enabled in their operating system accessibility settings.

Performance Budget
  • Homepage load time: Under 3 seconds on a 4G Nigerian connection. Google PageSpeed mobile score target: 75 or higher.
  • Images: Artisan profile photos served at exactly the size they are displayed (80×80px thumbnails for cards). Lazy-loaded — only load when the image is about to scroll into view.
  • Maps: Do not load the map on page load. Show a "Load map" button first. Only load the map JavaScript and tiles when the user clicks. This saves significant bandwidth.
  • Fonts: Load the system font first, then load the custom font asynchronously (non-blocking). Users see text immediately instead of waiting for a font to download.

  • Animations: CSS-only animations only on critical pages. No heavy JavaScript animation libraries on the homepage or booking flow.
STD-4

Design Handoff Standards

How to hand designs from Figma to developers correctly
Why This MattersA design that is not properly handed off is half-built. Developers need exact specifications for how things change at different screen sizes, what happens on hover, and how the page behaves when data is loading. This section defines what must be included for every screen before it is passed to engineering.
Figma File Organisation
  • Separate Figma file per portal: Nexplumb_Web_Customer · Nexplumb_Web_Artisan · Nexplumb_Web_Admin · Nexplumb_Web_Supplier. Keeping them separate prevents files becoming unmanageably large.
  • Three frames per screen: Every screen must exist as three separate frames side by side: "[Screen ID] Desktop 1280px", "[Screen ID] Tablet 768px", "[Screen ID] Mobile Web 375px". Engineers need all three to build responsively.
  • Frame naming convention: Always use the Screen ID from this document. E.g.: "CW-2 Search Results / Desktop 1280".
  • Auto Layout throughout: All frames and component groups must use Figma Auto Layout (not fixed-position). This translates to CSS Flexbox/Grid in the codebase and ensures designs are actually buildable.
Mandatory Annotations Per Screen
  • Breakpoint behaviour notes: Written annotations on the frame: "At 768px: left filter sidebar collapses into a drawer opened by a filter button." Never assume the developer knows what to do at different sizes.
  • Hover states: Every interactive element (buttons, cards, links, nav items) must have its hover state annotated or shown in an adjacent frame variant. "On hover: card shadow deepens, border changes to teal."
  • Sticky behaviour: "This navbar is position:sticky top:0. Its z-index must be higher than the map layer below." Developers need explicit instructions for sticky elements.
  • Animations: "Artisan map pin transition duration: 0.5 seconds ease. Position updates every 10 seconds." Be specific — "smooth" is not a specification.
  • Grid specification: State which grid template applies at each breakpoint — e.g. "Desktop: CSS Grid 12 columns, 24px gutters. Mobile: 4 columns, 16px gutters."
  • API dependencies note: "This screen requires: /api/artisans/search (with filters), /api/user/location (GPS)." This helps engineers understand what backend work is needed before the screen can be built.
  • Real-time requirements: "This screen updates live via WebSocket. Artisan pin position updates every 10–15 seconds." Makes it clear to engineering that a persistent connection is needed.
  • Third-party integrations: List any external services used: "Paystack (payments) · Google Maps API (map + pins) · NIMC API (NIN verification) · Twilio (WhatsApp messages) · Termii (SMS OTP)."
